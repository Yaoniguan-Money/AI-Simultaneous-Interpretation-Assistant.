# AI 同声传译助手

用户经常需要观看英语（或其他外语）演讲、技术分享、国际会议或网课，请开发一款 AI 同声传译助手，帮助用户降低语言门槛，提升信息获取效率。

通过 AI 能力，将单向音频流实时、流畅地翻译成中文，以字幕形式呈现，帮助用户跟上内容节奏。系统具备修正能力，能够自动纠正之前识别或翻译的错误。

## 双通道流水线架构

应用核心是一条两路并行的翻译流水线，分别解决"快"和"准"两个问题：

### Channel 1 —— 快通道（实时翻译）

负责端到端的低延迟翻译，数据流为：

```
音频 PCM → ASR 语音识别 → 语义分句 → LLM 流式翻译 → 字幕渲染
```

每一步的设计决策：

- **ASR 识别**：WebSocket 长连接，`connect()` 不可重入，通过连接预热机制在用户操作弹窗等待期间并行建连。interim（中间结果）直接推送到字幕层展示识别进度，final（确认结果）才送入分句器，避免不稳定文本污染翻译。
- **语义分句**：多策略切分——句末标点（`. ! ?`）立即切分，话题转换词（`So`, `Now`, `But` 等）辅助切分，静音间隔超过 800ms 触发切分，连续无标点超过 4s 强制交付。分句器只消费 final 结果，interim 不参与切句。
- **LLM 流式翻译**：SSE 流式接收 token，逐 token 写入字幕 atom，实现打字机效果。翻译请求携带共享上下文（领域、术语表、近期摘要）和最近 5 句翻译历史，让 LLM 感知语境。
- **强制交付防憋字**：ASR 长时间只发 interim 不发 final 时（说话人连续输出无停顿），取最新 interim 文本作为伪 final 强行送入分句器，避免字幕长时间空白。

### Channel 2 —— 慢通道（上下文理解）

异步运行，不阻塞 Channel 1。每积累 3 个完整句子触发一次 LLM 分析：

- **领域检测**：判断当前对话主题（技术讨论、商务会议、学术演讲等），附带置信度
- **术语提取**：识别专业术语及其对应翻译，注入 Channel 1 后续翻译的 system prompt
- **滚动摘要**：用最近 5 轮分析结果维持滑动窗口，避免上下文窗口膨胀
- **话题切换判断**：检测到话题转变时通知 Channel 1 清空翻译记忆，避免旧语境干扰新话题翻译

两个 Channel 通过 Jotai atoms 通信——Channel 2 写入 `sharedContextAtom`，Channel 1 每次翻译时读取，不经过 IPC，纯内存响应式。

### 修正引擎

基于上下文滚动修正：每翻译 3 句，回溯检查最早 1 句。当后文语境（术语、指代、话题）推翻前文翻译时，LLM 产出修正并通知字幕层触发 "✎ 根据上下文已修正" 动画。已修正过的句子不再重复触发。

### 会议纪要生成

停止翻译时，LLM 基于完整翻译历史生成结构化会议纪要——包含会议主题、关键议题（2-5 个）、每项议题的讨论要点、决策事项、行动项及负责人、一段话摘要。Promise 链编排保证顺序：先 flush 分句器缓冲完成最后几句翻译，再生成纪要，最后 dispose 资源。30 秒超时兜底防止 LLM 无响应卡死。

## 🚀 使用指南

### 第一步：配置 API 密钥

1. 启动应用，点击主界面「API 设置」
2. ASR 配置区选择语音识别供应商并填入凭证
3. LLM 配置区选择翻译模型供应商并填入 API Key
4. 点击「测试连接」验证

### 第二步：选择音频源

- **系统音频**：捕获电脑播放的声音（在线会议、视频翻译）
- **麦克风**：捕获麦克风输入（现场对话翻译）

### 第三步：开始翻译

1. 点击「▶ 开始翻译」
2. 系统音频模式下弹出屏幕选择对话框
3. 字幕悬浮窗自动弹出，翻译结果实时显示
4. 勾选「双语字幕」同步显示英文原文

### 第四步：查看历史

- 停止翻译后，「翻译历史」展示完整记录及 AI 会议纪要
- 支持复制全部到剪贴板或导出 Markdown

### 演示模式

无需 API 配置即可体验——内置预录制视频和预计算字幕，展示打字机流式输出和上下文修正动画的完整效果。

## 📁 项目结构

```
├── electron/                # Electron 主进程
│   ├── main.ts              # 窗口管理、IPC 处理、凭证加密存储

│   └── overlay-window.ts    # 透明悬浮窗工厂
├── src/
│   ├── components/
│   │   ├── layout/          # 主控制窗口 + 悬浮窗布局
│   │   ├── settings/        # ASR/LLM 配置面板
│   │   ├── subtitle/        # 字幕渲染（堆叠、逐行、修正标记）
│   │   ├── history/         # 翻译历史 + 会议纪要卡片
│   │   └── demo/            # 演示模式播放器
│   ├── services/
│   │   ├── asr/             # ASR 接口定义 + 4 供应商实现
│   │   ├── llm/             # LLM 接口定义 + OpenAI 兼容实现
│   │   └── pipeline/        # 快/慢通道管线 + 分句器 + 修正引擎
│   ├── hooks/               # 翻译会话、音频捕获、订阅同步等 Hook
│   ├── stores/              # Jotai atoms（设置、会话、共享上下文）
│   ├── utils/               # 音频环形缓冲、加密工具、延迟追踪
│   └── types/               # TypeScript 类型定义
├── shared/
│   ├── ipc-channels.ts      # IPC 通道名称常量
│   └── app-config.ts        # 应用全局配置常量
└── scripts/

    └── generate-icon.ps1    # 图标生成脚本
```
